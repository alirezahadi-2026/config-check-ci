# config-check-ci 
Manuelle Schritte ohne Automatisierung:
Datei prüfen
Syntax kontrollieren
Test manuell ausführen
Was ist CI?

Automatische Prüfung nach jedem Push oder Pull Request.

Wo beginnt CD?

Wenn nach erfolgreicher Prüfung automatisch deployt wird.

Vorteile:
Fehler werden früh erkannt.
Weniger manuelle Arbeit.
Mehr Sicherheit bei Änderungen.
Best Practices:
Kleine Änderungen committen.
Klare Commit-Nachrichten.
Schnelle Prüfungen.
